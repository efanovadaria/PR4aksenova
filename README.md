## Функциональные возможности

### Главная Страница
* Реализовали так, чтобы сразу открывалать первая страница с первой функцией.

### Страница с первой функцией
* Ввод: x, y, z
* Отображение скриншота формулы из вышеупомянутой таблицы (№1)
* Кнопка "Вычислить" - расчёт функции с помощью введенных значений и их вывод в поле ответа
* Кнопка "Очистить" - очищение полей ввода и ответа
* Кнопка "Далее" - переход на следующую страницу со второй функцией

### Страница со второй функцией
* Ввод: x, y
* Отображение скриншота формулы из вышеупомянутой таблицы (№2)
* 3 кнопки переключателя для выбора функции f(x)
  * sh(x)
  * x<sup>2<sup/>
  * e<sup>x<sup/>
* Кнопка "Вычислить" - расчёт функции с помощью введенных значений, выбора функций и их вывод в поле ответа
* Кнопка "Очистить" - очищение полей ввода и ответа
* Кнопка "Далее" - переход на следующую страницу с третьей функцией

### Страница с третьей функцией
* Ввод: x<sub>0</sub>, x<sub>k</sub>, dx, a, b
* Отображение скриншота формулы из вышеупомянутой таблицы (№3)
* Кнопка "Вычислить" - вычисление значений y(x) на отрезке ( x<sub>0</sub> - x<sub>k</sub> ) , а также их вывод в поле ответа и построение графика функции
* Кнопка "Очистить" - очищение полей ввода и ответа
* Кнопка "Назад" - возврат на главную страницу

## Вывод о проведённом тестировании

В проекте `UnitTestProject1` выполнено модульное тестирование: запущены 4 теста — демонстрационный тест на работу методов `Assert` и 3 базовых теста для автоматизированной проверки трёх математических функций ПР4 (формулы со страниц 1/2/3, вынесенные в `PR4aksenova.MathFunctions`).

## Причина успешного выполнения тестов

Тесты проходят успешно, потому что:
- проект с тестами содержит ссылку на основной проект (`ProjectReference` на `PR4aksenova`), поэтому методы `MathFunctions` доступны;
- код компилируется без ошибок;
- для заранее заданных входных данных вычисляется ожидаемое значение и сравнивается с фактическим результатом метода;
- для чисел типа `double` используется сравнение с допустимой погрешностью (`Assert.AreEqual(expected, actual, delta)`), что корректно для вещественных вычислений.
